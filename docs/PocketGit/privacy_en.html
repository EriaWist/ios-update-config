PocketGit Privacy Policy

Last updated: June 10, 2026

1. Overview

PocketGit is a Git repository management app designed for iPhone. We value your privacy. This Privacy Policy explains how PocketGit handles data and what you should know when using the app.

PocketGit does not actively collect, track, sell, or share your personal data.

2. Data We Collect

PocketGit does not send your personal data to our servers. The app does not use advertising trackers or third-party analytics tools.

Data created or used inside the app, such as Git repositories, commit information, branch information, remote settings, app settings, and related files, is primarily stored locally on your device.

3. Local Data Storage

To provide Git repository management features, PocketGit may store the following data locally on your device:

  • Git repositories added or created by the user
  • Repository files, commits, branches, remotes, and other Git-related information
  • App preferences and operation states
  • Git remote connection information entered or configured by the user
  • Credentials or access tokens saved by the user on the device

This data is used to provide app functionality and is not actively uploaded to our servers by PocketGit.

4. Third-Party Git Services

Users may choose to use PocketGit with GitHub or other third-party Git services. When you configure a remote, sign in to a third-party service, clone, fetch, pull, or push, related data may be transmitted directly between your device and the selected third-party service.

The handling of such data is governed by the privacy policies and terms of service of those third-party services. PocketGit does not control how third-party services process your data.

5. Credentials and Access Tokens

If you choose to use an access token, password, or other credential for GitHub or another Git service, PocketGit may store that credential locally on your device for future Git operations.

PocketGit does not actively send these credentials to our servers. Credentials are only used when you perform Git operations that require authentication with the third-party Git service you selected.

6. Git Server Repo Mode

PocketGit may provide Git Server Repo Mode, which allows your iPhone to provide repository access to other devices on the same Wi-Fi network.

When this feature is enabled, your device may make a repository accessible on your local network. Please enable this feature only on trusted networks and turn it off when it is no longer needed.

7. Analytics, Advertising, and Tracking

PocketGit does not use advertising SDKs, does not track users across apps or websites, and does not sell user data to third parties.

If we add crash reporting, analytics, or other data collection features in the future, we will update this Privacy Policy and disclose the relevant data usage in accordance with App Store requirements.

8. Data Sharing

PocketGit does not share, sell, or rent your personal data to third parties.

Repository data is only transmitted to a Git service when you intentionally configure a third-party remote and perform a Git operation such as push, pull, fetch, or clone.

9. Data Deletion

You can delete repositories, settings, or credentials inside the app. You may also delete the PocketGit app from iOS to remove local data stored by the app on your device.

If data has already been pushed to GitHub or another third-party service, deletion of that data must be handled through that third-party service.

10. Children’s Privacy

PocketGit is not specifically directed to children. We do not knowingly collect personal data from children.

11. Changes to This Privacy Policy

We may update this Privacy Policy from time to time. If significant changes are made, we will update the “Last updated” date on this page.
